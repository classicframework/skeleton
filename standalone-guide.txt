Classic Framework Skeleton Standalone Package Guide

1.1. Create the classicframework/core directory in the project root.
1.2. Create the classicframework/database directory in the project root.
1.3. Create the classicframework/migration directory in the project root.

2.1. Copy the /src folder and all files from the core repository into the skeleton classicframework/core directory.
2.2. Copy the /src folder and all files from the database repository into the skeleton classicframework/database directory.
2.3. Copy the /src folder and all files from the migration repository into the skeleton classicframework/migration directory.

3.1. Remove "classicframework/core" from the "require" section in composer.json.
3.2. Remove "classicframework/database" from the "require" section in composer.json.
3.3. Remove "classicframework/migration" from the "require" section in composer.json.

4. Add:
   "replace": {
     "classicframework/core": "self.version",
     "classicframework/database": "self.version",
     "classicframework/migration": "self.version"
   }

5. Extend autoload:
   "autoload": {
     "psr-4": {
       "app\\": "app/",
       "classicframework\\core\\": "classicframework/core/src/",
       "classicframework\\database\\": "classicframework/database/src/",
       "classicframework\\migration\\": "classicframework/migration/src/"
     }
   }

6. Add a file named STANDALONE (just a simple marker).

Result: a self-contained package that works without Composer, but still works if composer install is run.
